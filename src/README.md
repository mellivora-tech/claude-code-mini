# Claude Code 架构脚手架

本目录用文件结构复刻 Claude Code 的架构。每个子目录对应架构里的一层 / 一个机制，
目前为**纯占位**（只有 README 说明职责），尚无实现代码。

来源：`Claude Code Architecture.html`，从三个角度拆解同一套系统。

## 01 · 内部职责分层（系统在做什么）

从模型决策到界面展示，七层骨架，外加一条贯穿其中的扩展插槽。

| 层 | 目录 | 职责 |
| --- | --- | --- |
| 01 模型层 | [`model/`](model/) | 纯语言模型，无状态。收对话历史 + 工具定义，返回文本或工具调用请求。 |
| 02 工具层 | [`tools/`](tools/) | 模型唯一能碰真实世界的接口：读写文件、执行 bash、搜索代码。 |
| 03 编排层 | [`agent/`](agent/) | 「发请求 → 工具调用 → 本地执行 → 结果回填 → 再发请求」的状态机。 |
| 04 上下文管理层 | [`context/`](context/) | 历史摘要压缩、工具输出截断，解决「有限窗口装无限信息」。 |
| 05 权限层 | [`permission/`](permission/) | 横切于工具层之上的拦截机制，危险操作需用户确认。 |
| 06 提示工程层 | [`prompt/`](prompt/) | 用自然语言「编程」前面所有层的行为。 |
| 07 界面层 | [`interface/`](interface/) | 终端 UI、流式输出、diff 展示。 |

横切层：[`extension/`](extension/) —— 扩展 / 注册层。不是「第八层」，而是贯穿全栈的设计原则：
核心层天生可插拔，先注册轻量「目录条目」，真正内容延迟到被需要时才加载。

## 02 · 记忆（三种时间尺度）

见 [`memory/`](memory/)。「记忆」不是单一机制，而是三套独立系统：
会话内 / 项目（CLAUDE.md）/ 跨会话（RAG）。

## 03 · 速查表

| 用户想做什么 | 对应模块 | 操作方式 |
| --- | --- | --- |
| 读改代码、跑测试 | 对话 + 内置工具 | 直接说需求 |
| 接入外部系统 | MCP | 配置 MCP server |
| 学会做特定领域操作 | Skill | 放一个 SKILL.md |
| 记住项目规范背景 | 项目记忆 | 写 CLAUDE.md |
| 记住跨项目个人偏好 | 跨会话记忆 | 开关 + 被动纠正 |
| 控制危险操作权限 | 权限层 | 配置允许 / 禁止列表 |
| 看清它每步在做什么 | 界面层 | 直接观察终端输出 |

## 一句话总结

模型只负责决策，工具只负责执行，编排把两者串成循环；
MCP 给系统装新的「手」，Skill 给系统递一本新的「说明书」；
会话记忆是「这次没忘」，项目记忆是「这个项目得记住」，跨会话记忆是「上次聊的事还认得」。
